# OmarchyOS Knowledge Base

Omarchy = opinionated Arch Linux + Hyprland setup by DHH (Basecamp). "Omakase computing" — strong curated defaults, minimal necessary configuration. Installed via ISO. Underneath it's vanilla Arch — pacman, yay, AUR all work normally.

---

## Core Architecture — The Two-Directory Model

This is THE most important concept.

| Directory | Owner | Rule |
|---|---|---|
| `~/.local/share/omarchy` | Omarchy upstream | NEVER edit directly. Changes will conflict with `omarchy-update`. |
| `~/.config` | Yours | All customization goes here. Safe across updates. |

**Override pattern:** If you need to change something Omarchy owns in `~/.local/share/omarchy`, find the corresponding file in `~/.config` and override it there. Never fork the source.

If you must edit `~/.local/share/omarchy` directly, commit changes first or `omarchy-update` will fail/conflict.

## Key Config Files in `~/.config`

| File | Controls |
|---|---|
| `~/.config/hypr/hyprland.conf` | Main Hyprland config. Sources other files below. |
| `~/.config/hypr/bindings.conf` | All keybindings. Edit here, not hyprland.conf. |
| `~/.config/hypr/monitors.conf` | Monitor resolution, position, scale (`GDK_SCALE`). |
| `~/.config/hypr/looknfeel.conf` | Rounding, gaps, borders, visual style. |
| `~/.config/hypr/input.conf` | Keyboard options (compose key, kb_options). |
| `~/.config/hypr/hypridle.conf` | Idle/sleep behaviour. |
| `~/.config/hypr/hyprlock.conf` | Lock screen — symlinked to active theme. |
| `~/.config/waybar/config.jsonc` | Waybar layout. Edit freely. |
| `~/.config/waybar/style.css` | Waybar CSS — **symlinked to active theme**. Changes overwritten on theme switch. |
| `~/.config/walker/config.toml` | App launcher (Walker) config. |
| `~/.config/alacritty/alacritty.toml` | Terminal config. |
| `~/.config/uwsm/default` | Default `$EDITOR` and terminal. Requires Hyprland restart after change. |
| `~/.config/omarchy/themes/` | Your custom themes live here. |
| `~/.bashrc` | Shell aliases, functions, exports. **Never overwritten by updates.** Safe place for all shell customization. |
| `~/.XCompose` | Emoji and text autocompletion. Run `omarchy-restart-xcompose` after editing. |

**Access any config fast:** `Super + Alt + Space` → Setup → Configs → [process]. Auto-restarts the process after `:wq`.

## Hyprland Config Hierarchy

```
~/.config/hypr/hyprland.conf
  sources → bindings.conf     (keybindings)
  sources → monitors.conf     (display setup)
  sources → looknfeel.conf    (visual tweaks)
  sources → input.conf        (keyboard/mouse)
  sources → hypridle.conf     (idle)
```

Best practice for Rubait: append `source = ~/.config/hypr/custom.conf` to `hyprland.conf` and put all personal overrides in `custom.conf`. That way upstream changes to `hyprland.conf` don't eat your customizations.

**After any Hyprland config change:** Run `hyprctl configerrors` to catch syntax issues before reloading. Reload with `hyprctl reload` or `Super + Esc` (restarts UWSM/Hyprland).

## Waybar — Rubait-Specific Rules

- Signals **7–10 are reserved by OmarchyOS**. Custom modules must use signal **11+**.
- `~/.config/waybar/config.jsonc` is yours — edit freely.
- `~/.config/waybar/style.css` is **theme-symlinked** — any direct CSS edits are wiped on theme switch. Strategy: add a secondary CSS `@import` at the bottom of the theme's CSS, or manage style in a file that isn't theme-managed.
- In `exec` and `on-click` fields: use `$HOME` not `~` — Waybar does not expand `~`.
- For click actions that need user input: use `omarchy-launch-floating-terminal-with-presentation`.
- Tray icons hidden by default behind expander. To show all: change `group/tray-expander` → `tray` in `config.jsonc`.

## Theme System

- 17 built-in themes. Switch via `Super + Ctrl + Shift + Space` or Omarchy Menu → Style → Theme.
- Themes live at `~/.local/share/omarchy/themes/` (upstream, read-only).
- **Custom themes:** Copy a theme to `~/.config/omarchy/themes/[theme-name]`. It auto-appears in the theme menu. Never modify themes in `~/.local/share/omarchy/themes/`.
- Main theme file: `colors.toml` — defines color set that auto-generates configs for Alacritty/Ghostty/Kitty, btop, Chromium, Hyprland, Hyprlock, Mako, SwayOSD, Walker, Waybar.
- Cycle wallpapers within theme: `Super + Ctrl + Space`.
- Obsidian theme must be set manually inside Obsidian (Appearance → Themes → Omarchy).
- Light mode: drop empty file `light.mode` in theme root.
- Icon color matching: add `icons.theme` file with Yaru variant name.
- Distribute theme: push to public git, install via Omarchy Menu → Install → Theme → [git URL]. Name as `omarchy-[themename]-theme`.

## GDK_SCALE — Critical for Rubait's Setup

OmarchyOS defaults to `GDK_SCALE=2` (assumes HiDPI/retina display).

**Rubait runs a standard 1080p/standard-DPI monitor on EliteDesk.** This makes GTK apps render oversized.

**Fix:**
```ini
# ~/.config/hypr/monitors.conf or hyprland.conf
env = GDK_SCALE,1
```
Also add to `~/.bashrc`:
```bash
export GDK_SCALE=1
```
Spotify: use `Ctrl + Minus` / `Ctrl + Plus` to manually adjust its zoom regardless of GDK_SCALE.

## Monitor Config (Multi-Machine Portable SSD Use)

```ini
# ~/.config/hypr/monitors.conf
# Specific monitor by name (run `hyprctl monitors` to get name):
monitor=HDMI-A-1,1920x1080@60,0x0,1

# Fallback for unknown/any monitor — ESSENTIAL for portable SSD use:
monitor=,preferred,auto,1
```

The fallback `monitor=,preferred,auto,1` ensures any machine Rubait boots on gets sane defaults automatically. Without it, unknown monitors may get no output.

## Updates

- **Always use:** `Update > Omarchy` in Omarchy Menu — pulls latest code + runs migrations + updates packages.
- **Never use:** `pacman -Syu` or `yay -Syu` directly — skips Omarchy migrations, can break configs.
- Update channels: `stable` (default) → `RC` → `edge` → `dev`. Switch via `Update > Channel`.
- Rollback: restart and pick the pre-update snapshot in the GRUB/systemd-boot menu.
- Nuclear reset: `omarchy-reinstall` — restores all default configs (overwrites your `~/.config` Omarchy changes).
- Per-config reset: `Update > Config` in Omarchy Menu.
- Debug: `omarchy-debug` — generates shareable diagnostic output.

## Shell Customization

`~/.bashrc` is the safe zone. Add aliases, functions, exports here. Never overwritten by updates.

Omarchy's default shell is **bash**. (zsh presence = legacy leftover from old installs — `.zshrc` is safely deletable if present from a previous setup).

## Common Tweaks

**Rounded corners** (`~/.config/hypr/looknfeel.conf`):
```ini
decoration {
    rounding = 8
}
```

**Remove window gaps:**
```ini
general {
    gaps_in = 0
    gaps_out = 0
    border_size = 0
}
```

**Toggle top bar:** `Super + Shift + Space`

**Caps Lock → Compose key** (default in Omarchy). To swap Caps Lock ↔ Esc instead:
```ini
# ~/.config/hypr/input.conf
kb_options = caps:swapescape
```
To use right alt as compose instead:
```ini
kb_options = compose:ralt
```

## Troubleshooting Quick Reference

| Problem | Fix |
|---|---|
| Apps too large on screen | Set `env = GDK_SCALE,1` in `monitors.conf` |
| Hyprland config broken | `hyprctl configerrors` to diagnose; `omarchy-reinstall` nuclear option |
| Locked out (wrong password) | `Ctrl+Alt+F2` → login as root → `faillock --reset --user [username]` |
| Update broke something | Reboot → select pre-update snapshot in bootloader |
| Caps Lock not working | Remapped to XCompose by default — see `input.conf` |
| `omarchy-update` fails | Check for uncommitted changes in `~/.local/share/omarchy` |
| Speakers not working | Click speaker in Waybar → set as default with `d` in wiremix |
| Waybar module not updating | Check signal number — 7–10 reserved, use 11+ |
